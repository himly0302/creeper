设计一个爬虫工具, 传入一个文件格式如下

```markdown
# H1

## H2
https://url1
http://url2

```

1. 通过爬取网页内容, 可以知晓页面的 标题、简介、主题内容等

那么最终以如下形式保存文件
H1(文件夹名)
- H2(文件夹名)
  - title1.md(网页url1的标题)
  - title2.md(网页url2的标题)

在title1.md文件中 有 网页标题、简介、爬取时间、主题内容、来源链接等必要信息。


2. 文件保存成功后 会在redis中标记, 在爬取网页前会判断 该页面是否被爬取过, 如果被爬取过 则跳过
```
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=1
REDIS_PASSWORD=
```

3. 如果直接爬取网页失败，那么Playwright 动态采集； 自动降级
4. 对于需要登陆的网站，进行cookie管理，下次直接登陆
5. 内容清洗 过滤广告等无关内容
6. 支持并发处理(多线程还是啥技术，你自己决定)，提高执行效率
7. 有debug模型，开启会会在执行中打印出详细的操作日志信息
8. 考虑对失败情况做兼容处理<特别是反爬虫错误>


注意：
1. redis 在全局中已经安装好了，本项目只需要连接使用
2. 使用python的虚拟环境，避免污染全局
