设计一个爬虫工具, 传入一个文件格式如下

```markdown
# H1

## H2
https://url1
http://url2

```

1. 通过爬取网页内容, 可以知晓页面的 标题、简介、主题内容等

那么最终以如下形式保存文件
H1(文件夹名)
- H2(文件夹名)
  - title1.md(网页url1的标题)
  - title2.md(网页url2的标题)

在title1.md文件中 有 网页标题、简介、爬取时间、主题内容、来源链接等必要信息。


2. 文件保存成功后 会在redis中标记, 在爬取网页前会判断 该页面是否被爬取过, 如果被爬取过 则跳过
```
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=1
REDIS_PASSWORD=
```

3. 如果直接爬取网页失败，那么Playwright 动态采集； 自动降级
4. cookie管理
```
使用 Playwright 打开浏览器，录入登录凭证
打开浏览器，让用户手动登录，然后提取 Cookie 保存到 Redis; 后续此网站访问时，直接从 Redis 中读取 Cookie
流程如下 => 打开浏览器加载要登陆的网址， 在浏览器中手动登录， 登录完成后，关闭浏览器窗口即可自动保存 Cookie
```
5. 内容清洗 过滤广告等无关内容
6. 支持并发处理(多线程还是啥技术，你自己决定)，提高执行效率
7. 有debug模型，开启会会在执行中打印出详细的操作日志信息
8. 考虑对失败情况做兼容处理<特别是反爬虫错误>


注意：
1. redis 在全局中已经安装好了，本项目只需要连接使用
2. 使用python的虚拟环境，避免污染全局
3. test测试文件存放在tests文件夹下
4. 生成一个sh文件，清空redis中关于本项目的内容，删除所有已下载的文件；便于测试使用
5. 在 读取网页内容后，发现有些网页的 标题、summary、正文 都是英文， 先让大模型先给他翻译成中文，然后在进行保存
```
OPENAI_BASE_URL=https://api-inference.modelscope.cn/v1/ 
LLM_MODEL=deepseek-ai/DeepSeek-V3.1 
OPENAI_API_KEY=
```
